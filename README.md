# Kanban

Kanban — это Java‑приложение (без фреймворков), реализующее трекер задач в стиле «канбан‑доски».  
Приложение позволяет создавать задачи разных типов, управлять их жизненным циклом, хранить историю просмотра и сохранять данные в разных хранилищах (память, файл, сервер).

## Функциональность

### Типы задач

В системе есть три типа задач:

- **Task** — обычная задача.  
- **Epic** — большая задача, объединяющая несколько подзадач.  
- **Subtask** — подзадача, входящая в Epic.

Для всех типов задач доступны CRUD‑операции:

- создание, чтение, обновление и удаление задач;  
- получение всех задач определённого типа;  
- получение подзадач конкретного Epic.

### История просмотра задач

Приложение ведёт историю последних просмотренных задач:

- просмотр задачи добавляет её в историю;  
- повторные просмотры не дублируются — задача перемещается в конец истории;  
- история используется, чтобы быстро вернуться к актуальным задачам.

### Приоритет и пересечение задач во времени

У задач могут быть время начала и продолжительность:

- задачи сортируются по приоритету (по времени начала);  
- реализована проверка пересечений во времени, чтобы две задачи не занимали один и тот же интервал.

## Варианты хранения данных

Функциональность реализована в нескольких вариантах хранения:

1. **In‑memory** — хранение задач в оперативной памяти.  
2. **File‑backed** — сохранение задач и истории в файл (например, в CSV) и восстановление состояния при старте.  

## Архитектура

Проект построен вокруг менеджера задач:

- интерфейс `TaskManager` с операциями над задачами, эпиками, подзадачами и историей;  
- реализации `InMemoryTaskManager`, `FileBackedTaskManager`;  
- отдельный `HistoryManager`, отвечающий за хранение истории просмотров.

Основные сущности:

- `Task` — базовый класс задачи;  
- `Epic` — расширение задачи с набором `Subtask`;  
- `Subtask` — задача, ссылающаяся на свой `Epic`;  
- перечисления для статуса задач (`NEW`, `IN_PROGRESS`, `DONE`) и, при необходимости, для типа.

Отдельные классы/компоненты отвечают за:

- работу с файлами (чтение/запись CSV);  
- работу с HTTP‑сервером/клиентом;  
- сериализацию/десериализацию JSON (через Gson).

## Стек технологий

Проект демонстрирует работу с Java Core и сопутствующими библиотеками:

- Java 11+ (коллекции, generics, OOP, исключения);  
- JUnit 5 — модульное тестирование;  
- Gson — сериализация/десериализация задач в JSON;  
- Встроенный HTTP‑сервер/клиент из стандартной библиотеки Java;  
- Работа с файлами (`java.io`, `java.nio.file`).

## Запуск проекта

Так как это учебный проект без фреймворков, запуск обычно делается из `main`‑класса:

1. Открыть проект в IntelliJ IDEA.  
2. Убедиться, что настроен JDK 11+.  
3. Найти класс с методом `public static void main(String[] args)` (например, `Main` или `HttpTaskServer`) и запустить его.
